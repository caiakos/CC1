BEGIN TRANSACTION

Inicia uma transação no banco de dados.

BEGIN TRANSACTION [WITH CONSISTENT SNAPSHOT];

Imagine que você está fazendo várias compras em uma loja. Antes de pagar, você inicia uma transação, o que significa que está prestes a fazer várias operações de compra.

MySQL, PostgreSQL, Oracle DB, SQL Server, SQLite

COMMIT

Confirma as alterações feitas durante a transação e as torna permanentes.

COMMIT;

Depois de terminar suas compras e passar no caixa, você paga e as compras são confirmadas.

MySQL, PostgreSQL, Oracle DB, SQL Server, SQLite

ROLLBACK

Desfaz as alterações feitas durante a transação e restaura o estado anterior.

ROLLBACK [TO SAVEPOINT ponto1];

Imagine que, antes de pagar, você percebe que pegou o item errado. Você decide cancelar todas as suas compras.

MySQL, PostgreSQL, Oracle DB, SQL Server, SQLite

SAVEPOINT

Define um ponto dentro de uma transação para o qual você pode voltar posteriormente.

SAVEPOINT ponto1;

Imagine que você está fazendo uma viagem longa de carro com alguns amigos. Em determinado ponto da viagem, você cria um "ponto de parada" (SAVEPOINT) em uma cidade específica. Isso permite que, se algo der errado mais tarde na viagem, você possa voltar para esse ponto de parada e recomeçar a partir dali, sem ter que voltar ao início da viagem.

PostgreSQL, Oracle DB

RELEASE SAVEPOINT

Remove um SAVEPOINT previamente definido.

RELEASE SAVEPOINT ponto1;

Após completar com sucesso uma parte da sua viagem e já não precisar mais do ponto de parada em uma cidade específica, você decide liberar esse ponto de parada para economizar tempo e recursos. Isso é semelhante ao RELEASE SAVEPOINT no SQL, onde você remove um ponto de parada que já não é mais necessário na transação.

PostgreSQL, Oracle DB

SET TRANSACTION

Define atributos de transação, como nível de isolamento ou modo de acesso.

SET TRANSACTION ISOLATION LEVEL SERIALIZABLE [READ WRITE];

Ao iniciar sua viagem de carro, você pode escolher diferentes configurações para a viagem, como escolher a estrada mais rápida (nível de isolamento) ou decidir se quer parar em todas as atrações turísticas ao longo do caminho (modo de acesso). O comando SET TRANSACTION no SQL permite que você defina essas configurações para a sua transação, como o nível de isolamento da transação ou o modo de acesso aos dados.

MySQL, PostgreSQL, Oracle DB, SQL Server, SQLite
